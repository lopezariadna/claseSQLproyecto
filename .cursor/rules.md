# Reglas del proyecto

Este proyecto tiene fines educativos.

Siempre explicar antes de generar código.

No crear archivos innecesarios.

No usar patrones avanzados sin explicarlos primero.

No modificar código existente sin explicar el motivo.

Priorizar código simple y fácil de entender.

Cuando exista más de una forma de hacer algo, explicar las ventajas y desventajas antes de elegir una.

Asumir que el desarrollador está aprendiendo desde cero.
